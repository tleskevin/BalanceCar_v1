# STM32F103C8T6 雙模控制小車 (BalanceCar_v1 & LineTracking_v1)

本專案基於 **STM32F103C8T6 (Blue Pill)** 核心板，整合了 **PID 級聯雙輪自平衡系統** 與 **大扭力爬坡循跡系統**。專案針對物理重心不對稱、齒輪箱死區摩擦及上坡動能損失進行了深度軟體優化。

---

## 🚀 核心功能

### 1. 雙輪自平衡模式 (Balance Mode)
* **級聯 PID 架構**：內環（直立）PD 穩定姿態，外環（速度）PI 消除位置漂移。
* **防暴衝機制**：具備抗積分飽和 (Anti-Windup)，徹底根除因重力導致的積分無限累積，防止車體失控甩飛。
* **動態零點校準**：引入 `BALANCE_OFFSET_ANGLE` 巨集，輕鬆適應加裝配件後的物理配重偏移。

### 2. 強扭力循跡爬坡模式 (Line Tracking & Hill Climb Mode)
* **比例修正演算法**：利用動態 PWM 偏移量實現平滑轉彎，解決大彎道過彎半徑不足的問題。
* **爬坡扭力補償**：透過拉高 `SPEED_BASE` 與設定 `PWM_MIN_LIMIT`，確保車輛在爬坡時克服重力，不會因為失去轉速而停滯。
* **即時扭力響應**：優化 `Ramp_PWM` 斜率控制，大幅提升步進數值 (`step = 30`)，確保馬達在負載變化時能瞬間輸出扭力。

## 🛠️ 硬體週邊與引腳分配

| 週邊功能 | MCU 引腳 (Pin) | 實體連接說明 |
| :--- | :--- | :--- |
| **主控板** | STM32F103C8T6 | 核心時脈 72MHz (HSE 8MHz + PLL x9) |
| **陀螺儀 (I2C1)** | PB8 (SCL), PB9 (SDA) | MPU6050 (VCC朝車尾) |
| **馬達 PWM (TIM2)** | PA2 (CH3), PA3 (CH4) | PA2 -> 右輪 PWM / PA3 -> 左輪 PWM |
| **循跡感測器** | PB0 (左), PB1 (右) | 紅外線感測器，低電位觸發 (黑線) |
| **馬達方向 (GPIO)** | PB12~PB15 | PB12/13: 左馬達 / PB14/15: 右馬達 |
| **編碼器 (TIM3/4)** | PA6/7, PB6/7 | 雙通道正交解碼 |

## ⚙️ 關鍵參數調校指南 (main.c)

透過 `main.c` 頂部的 `#define` 巨集即可快速適應不同環境：

### 循跡爬坡模式：
* `PWM_MIN_LIMIT (80)`：克服齒輪箱摩擦力的啟動底線。
* `SPEED_BASE (220)`：基礎巡航動力（爬坡建議調至 300~350）。
* `SPEED_CORRECT_FAST (300)`：轉彎外側輪爆發力，數值越大，轉彎越急。
* `SPEED_CORRECT_SLOW (-50)`：轉彎內側輪輔助，設為負值可啟用坦克迴旋。

### 平衡模式：
* `BALANCE_OFFSET_ANGLE`：校準重力中立點，若小車緩慢移動，請微調此參數。
* `SPEED_ANGLE_MAX`：限制速度環輸出，數值越小移動越穩。

## 💻 開發環境
* **IDE**：Keil uVision 5 (Compiler V5.06)
* **硬體設定**：STM32CubeMX (HAL 庫)
* **編譯注意**：
    1. 開啟 `.uvprojx` 專案檔。
    2. 確認 Include 路徑已包含 `/Inc` 目錄。
    3. 燒錄前確認 `SystemClock_Config` 已鎖定為 72MHz。
